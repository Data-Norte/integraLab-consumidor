Painel publico de QA para acompanhar webhooks, resultados e PDFs gerados

Esta tela replica o conceito de "Logs de integracao" do console, mas focada no consumidor de testes. Cada item processado gera arquivos locais por tenant, registra o payload recebido da API, o resultado devolvido e disponibiliza o PDF gerado para o QA validar o fluxo completo.

Use os atalhos acima para gerar um novo lote QA ou processar os exames pendentes do HML.

Objetivo desta pagina

Dar visibilidade real para o QA: o que entrou, o que saiu, qual tenant foi afetado, qual PDF foi montado e quais arquivos foram gravados localmente para auditoria tecnica.

Como testar
  1. Garanta que o `integraLab-api` esteja online e que o vinculo tenha webhook apontando para esta URL publica.
  2. Configure no consumidor `LAB_APOIO_VINCULO_ID`, `LAB_APOIO_AUTH_SECRET` e `LAB_APOIO_WEBHOOK_SECRET`.
  3. Deixe `AUTO_PROCESS_WEBHOOK=true` para fluxo automatico ou use `POST /api/lab-apoio/v1/consumer/processar-pendentes` para disparo manual.
  4. Abra esta pagina e filtre pelo tenant para acompanhar cada item processado.
O que precisa existir no Macro
  • Laboratorio de apoio ativo em `lab_apoio`, com `lab_uuid` consistente com a conta/vinculo.
  • Relacionamentos ativos em `lab_apoio_unidade` e `lab_apoio_unidade_exame` para os codigos de exame que o QA quer exercitar.
  • Itens pendentes em `agendaexameitens` com `liberado <> 'SIM'`, permitindo a API listar as pendencias.
  • Na API, variaveis do Macro configuradas para PDF: `LAB_APOIO_MACRO_PRF_ID`, `LAB_APOIO_MACRO_SERV_ID` e `LAB_APOIO_MACRO_FORNEC_ID`.
Consultas SQL de validacao 
SELECT id, descricao, lab_uuid, status
FROM lab_apoio
WHERE lab_uuid IS NOT NULL;

SELECT codexame, eh_ativo
FROM lab_apoio_unidade_exame
WHERE eh_ativo = 1;

SELECT id, codagendaexame, codexame, liberado
FROM agendaexameitens
WHERE COALESCE(liberado, 'NAO') <> 'SIM'
ORDER BY id DESC;
Endpoints disponiveis
  • GET /health: health geral do servidor.
  • GET /api/lab-apoio/v1/consumer/health: health do modulo consumidor.
  • POST /api/lab-apoio/v1/consumer/webhook: recebe webhook da API.
  • POST /api/lab-apoio/v1/consumer/processar-pendentes: dispara processamento manual.
  • GET /qa: abre esta tela publica de QA.
  • GET /api/lab-apoio/v1/consumer/qa/overview: resumo com KPIs, tenants, runs e itens.
  • GET /api/lab-apoio/v1/consumer/qa/itens/:itemId: detalhe completo do item processado.
  • GET /api/lab-apoio/v1/consumer/qa/itens/:itemId/arquivos/:kind: baixa ou visualiza entrada, saida, resultado, resposta ou pdf.
  • GET /api/lab-apoio/v1/consumer/qa/runs/:runId/arquivos/:kind: baixa contexto, webhook ou resumo da execucao.
  • DELETE /api/lab-apoio/v1/consumer/qa/storage: limpa SQLite e diretorios locais de QA.
